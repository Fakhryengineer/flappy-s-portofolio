🎯 UPLOAD TO GITHUB - STEP BY STEP:

┌─────────────────────────────────────────────────────────────┐
│  STEP 1: Initialize Git Repository                         │
└─────────────────────────────────────────────────────────────┘

git init

┌─────────────────────────────────────────────────────────────┐
│  STEP 2: Add All Files                                     │
└─────────────────────────────────────────────────────────────┘

git add .

┌─────────────────────────────────────────────────────────────┐

│  STEP 4: Rename Branch to Main                             │

   Jika diminta login, gunakan Personal Access Token
   (bukan password GitHub biasa)

   GitHub → Settings → Developer Settings → 
   Personal Access Tokens → Generate new token
   
   Permission yang diperlukan:
   ✓ repo (full control of private repositories)

1. Verifikasi di GitHub:
   - Semua file ter-upload
   - README.md display dengan baik
   - License terlihat

2. Update Repository Settings:
   - Add description
   - Add topics: portfolio, react, vite, tailwindcss
   - Add website URL (setelah deploy)

3. Deploy to Production:
   Option A: Vercel (Recommended)
   - Connect GitHub repo
   - Auto-deploy on push
   - Free subdomain: yoursite.vercel.app
   
   Option B: Netlify
   - Drag & drop dist folder
   - Or connect GitHub repo
   
   Option C: GitHub Pages
   - Run: pnpm add -D gh-pages
   - Update vite.config.ts base path
   - Run: pnpm deploy

Problem: "remote origin already exists"
Solution: git remote remove origin
          git remote add origin [URL]

Problem: "failed to push some refs"
Solution: git pull origin main --allow-unrelated-histories
          (then resolve conflicts if any)
          git push -u origin main

Solution: Use Personal Access Token instead of password

Solution: Check .gitignore includes node_modules/
